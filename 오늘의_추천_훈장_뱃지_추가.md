### 1. 훈장 뱃지 추가

- **테두리**: 황금색 (`border-amber-200`)

  - 크기: `text-xs`

- **위치**: 카드 우측 상단 (`absolute top-4 right-4`)
- **그림자**: `shadow-sm`

## 🎯 동작 방식

### 조건부 표시 로직
```
1. isTopRecommended === true (첫 번째 태스크)
   AND
2. isApplied === false (아직 지원하지 않음)
   ↓
   훈장 뱃지 표시
```

### 우선순위
- **"지원 완료" 뱃지** > **"매우 적합" 뱃지**
- 지원이 완료되면 "매우 적합" 뱃지는 숨겨지고 "지원 완료" 뱃지만 표시됨

### 어떤 태스크가 오든 첫 번째에 표시
```typescript
tasks.map((task, index) => (
  <TaskCard
    isTopRecommended={index === 0}  // 항상 index가 0인 것만 true
  />
))
```

- 데이터베이스에서 어떤 태스크가 조회되든 상관없이
- 오늘의 추천 업무 섹션의 **맨 위(index === 0)**에 있는 태스크에만 표시됨
- 동적으로 자동 적용

## 🎨 디자인 특징

### 색상 선택 이유
- **황금색 (Amber)**: 최고 추천, 우수성을 나타냄
- **부드러운 배경 (`amber-50`)**: 시니어 친화적, 눈에 부담 없음
- **진한 텍스트 (`amber-700`)**: 가독성 확보

### 아이콘 선택
- **훈장 (Medal)**: 직관적으로 "최고", "추천"을 나타냄
- **18x18px**: 작지만 명확하게 인식 가능

### 반응형 디자인
- 호버 시 카드가 확대되어도 뱃지는 고정 위치 유지
- 모바일/데스크톱 모두 동일하게 표시

## 🧪 테스트 시나리오

### 1. 정상 표시
- ✅ 오늘의 추천 업무 첫 번째 태스크에 훈장 뱃지 표시
- ✅ "매우 적합" 텍스트 표시

### 2. 지원 후
- ✅ 첫 번째 태스크 지원 시 훈장 뱃지 사라짐
- ✅ "지원 완료" 뱃지로 교체

### 3. 데이터 변경
- ✅ 다른 태스크가 첫 번째로 오면 해당 태스크에 훈장 표시
- ✅ 동적으로 자동 적용

### 4. 엣지 케이스
- ✅ tasks가 1개만 있어도 정상 표시
- ✅ tasks가 0개면 표시 안 함 (정상)
